---
name: Documentar antes de desarrollar
description: El usuario quiere construir documentación completa del estado deseado antes de escribir código
type: feedback
---

Cuando se discuten reglas de negocio, flujos o decisiones de arquitectura, solo documentar en CLAUDE.md.
No proponer implementación ni código hasta que el usuario lo pida explícitamente.

**Why:** El objetivo actual es construir una documentación que defina el estado deseado de la aplicación para que Claude Code pueda desarrollar de forma autónoma después.

**How to apply:** Al descubrir gaps en el código o decisiones de negocio, agregar la regla a la tabla BR en CLAUDE.md y describir el problema, pero no escribir código ni sugerir implementación a menos que el usuario lo solicite.
